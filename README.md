## Dog Store — Магазин товаров для собак

Учебный проект интернет-магазина товаров для собак, выполненный в рамках 6 практических занятий.
Отчёт оформлен в виде этого README и исходного кода в репозитории.

Реализованы:

- стилизация карточки товара с использованием SASS;
- backend на Node.js + Express с CRUD по товарам;
- тестирование API через Postman (по инструкции в проекте);
- frontend на React (Vite) с каталогом и корзиной;
- документирование API через Swagger.

---

### Практическое занятие №1 — SASS и карточка товара

**Требование:**  
Сверстать карточку товара (картинка, название, описание, цена, кнопка «В корзину»), оформить её с помощью SASS, используя переменные, миксины и вложенность селекторов.

**Реализация в проекте:**

- Папка `frontend/`.
- Файл `frontend/index.html`:
  - содержит статичную карточку товара с изображением, заголовком, описанием, ценой и кнопкой.
- Файл `frontend/styles.scss`:
  - использование переменных (`$primary-color`, `$secondary-color`, `$card-bg`, `$border-radius` и др.);
  - миксин `@mixin button-style(...)` для оформления кнопки;
  - вложенность `.product-card { &__image { ... } &__content { ... } &__title { ... } &__button { ... } }`.
- Файл `frontend/styles.css`:
  - скомпилированный CSS для подключения в HTML.
- В `package.json` настроены скрипты:
  - `npm run sass:build` — разовая компиляция SASS → CSS;
  - `npm run sass:watch` — автоматическая пересборка стилей при изменениях.

**Результат:**  
Красивая карточка товара, оформленная через SASS с использованием переменных, миксина и вложенности.

---

### Практическое занятие №2 — Backend на Node.js + Express (CRUD)

**Требование:**  
Создать серверное приложение на Node.js + Express с CRUD‑операциями над товарами:

- хранение списка товаров в памяти;
- эндпоинты:
  - `GET /products` — список товаров;
  - `GET /products/:id` — товар по id;
  - `POST /products` — создание;
  - `PATCH /products/:id` — частичное обновление;
  - `DELETE /products/:id` — удаление.

**Реализация в проекте:**

- Файл `server.js` в корне:
  - инициализация приложения Express;
  - массив `products` с товарами (`id`, `name`, `price`, `category`, `description`, `quantity`);
  - функция `getNextId()` для генерации нового идентификатора;
  - реализованы все указанные эндпоинты с обработкой ошибок (404, 400).
- В `package.json`:
  - скрипт `npm start`, запускающий `server.js`.

**Результат:**  
Работающий REST API по товарам магазина, возвращающий и изменяющий данные в формате JSON.

### Практическое занятие №3 — Тестирование API в Postman и внешний API

**Требование:**  

- протестировать собственный backend через Postman (несколько запросов разных типов);
- протестировать внешний публичный API и проанализировать ответы JSON.

**Реализация в проекте:**

- Файл `docs_postman.md`:
  - инструкция по выполнению запросов к своему API:
    - `GET /products`,
    - `POST /products` (с примером JSON‑тела),
    - `DELETE /products/:id`;
  - список примерных запросов к внешнему API Dog CEO (`https://dog.ceo/api/...`) с описанием ожидаемых JSON‑ответов.

**Практическая часть для отчёта:**  
По инструкции из `docs_postman.md` в Postman выполняются запросы, снимаются скриншоты запросов и ответов и прикладываются в отчёт/СДО.

### Практическое занятие №4 — Frontend (React) и связь с backend

**Требование:**  

- создать одностраничное приложение магазина на React;
- получать список товаров с backend;
- реализовать корзину (добавление, удаление, подсчёт суммы).

**Реализация в проекте:**

- Папка `client/` — React‑приложение на Vite.
- В `server.js`:
  - настроен CORS (`cors`) с разрешённым источником `http://localhost:5173`.
- Файл `client/src/App.jsx`:
  - при монтировании компонента выполняется запрос `GET /products` к backend;
  - данные сохраняются в состоянии `products` и отображаются в каталоге;
  - реализована корзина:
    - состояние `cartItems` для выбранных товаров;
    - функции `handleAddToCart`, `handleRemoveFromCart`, `handleClearCart`;
    - подсчёт общей суммы через `reduce` и `useMemo`.
- Файл `client/src/App.css`:
  - оформление основной страницы, сетка «каталог + корзина», карточек товаров и блока корзины.

**Результат:**  
Интерактивный frontend магазина: пользователь видит список товаров, может добавлять их в корзину, изменять количество и видеть общую сумму заказа.

### Практическое занятие №5 — Документация API (Swagger)

**Требование:**  

- документировать своё API с помощью Swagger;
- описать схему `Product` и все эндпоинты `/products`, `/products/{id}`;
- предоставить веб‑страницу документации.

**Реализация в проекте:**

- В `server.js`:
  - подключены библиотеки `swagger-jsdoc` и `swagger-ui-express`;
  - задан `swaggerDefinition` с информацией об API (название, версия, описание, сервер);
  - сформирован `swaggerSpec` и опубликован по адресу `/api-docs`;
  - через JSDoc‑комментарии описаны:
    - схема `Product` (тип объекта и его поля);
    - эндпоинты:
      - `GET /products`,
      - `GET /products/{id}`,
      - `POST /products`,
      - `PATCH /products/{id}`,
      - `DELETE /products/{id}`.

**Результат:**  
По адресу `http://localhost:3001/api-docs` доступна интерактивная Swagger‑документация, в которой можно посмотреть структуру запросов/ответов и отправлять запросы к API.

### Как запустить backend

```bash
cd Dog_Store
npm install        # при первом запуске
npm start
```

Сервер поднимется на `http://localhost:3001`.

**Swagger‑документация:**

- `http://localhost:3001/api-docs`

---

### Как запустить frontend (React)

```bash
cd Dog_Store/client
npm install        # при первом запуске
npm run dev
```

По умолчанию приложение будет доступно на `http://localhost:5173`.
